# ecomm-ref-payments Scope Overview

## Overview

The `ecomm-ref-payments` scope is a `reference`-type scope. It contains compliance standards and regulations that ecomm payment teams MUST adopt or map their implementations against. It does not document live systems — see `ecomm-plat-cloud` for live infrastructure.

Reference scopes are read-only blueprints. Teams use them as a source of truth when verifying compliance or designing new flows, not as operational runbooks.

## Content

This scope covers:

- PCI-DSS compliance requirements applicable to the ecomm platform.
- Payment industry regulations and mandatory security controls.
- Adoption guidance: how team scopes SHOULD reference and map against these standards.
